admin_bases module

admin_bases.py

Contiene todas las clases y métodos que crean y administran las bases de datos.

class admin_bases.BaseClientes[fuente]

Bases: Conexion

La clase BaseClientes() cuenta con los métodos que brindan con las directrices necesarias para administrar la información de la tabla «clientes».

alta_sql(cliente)[fuente]

Agrega un nuevo cliente en la tabla «clientes».

Parámetros:

cliente – (_tuple_) datos proporcionados desde el módulo modelo.py para la alta del cliente.

borra_cliente_sql(id_cliente)[fuente]

Borra al cliente seleccionado de la tabla «clientes».

Parámetros:

(_int_) (id_cliente) – id del cliente seleccionado.

borra_consumos_cliente_borrado_sql(id_cliente)[fuente]

Borra de la tabla «consumos» los datos correspondientes al cliente borrado con el método borra_cliente_sql().

Parámetros:

(_int_) (id_cliente) – id del cliente seleccionado.

cierra_sql(id_cliente)[fuente]

Modifica el atributo estado del cliente seleccionado en la tabla «clientes» y agrega horario de cierre al atributo hora_egreso del cliente seleccionado.

Parámetros:

id_cliente – (_int_) id de cliente.

estado_mesa_sql(id_cliente)[fuente]

Proporciona el atributo estado del cliente seleccionado.

Parámetros:

id_cliente – (_int_): id del cliente seleccionado

Returns True:

(_boolean_) si la mesa del cliente seleccionado está «ABIERTA»

Nota

Si el estado de la mesa del cliente seleccionado es «CERRADA», el método retorna una ventana de error.

modifica_comensal_sql(id_cliente, operacion)[fuente]

Modifica el atributo n_comensales del cliente seleccionado en la tabla «clientes».

Parámetros:

  • id_cliente – (_int_) ID del cliente seleccionado

  • operacion – (_str_) «suma» o «resta» según se vaya a agregar o quitar un comensal.

modifica_mesa_sql(nueva_mesa, id_cliente)[fuente]

Modifica el atributo n_mesa del cliente seleccionado en la tabla «clientes».

Parámetros:

  • nueva_mesa – (_int_): número de mesa proporcionado por el módulo modelo.py.

  • id_cliente – (_int_): ID del cliente seleccionado.

muestra_clientes_sql(estado1)[fuente]

Obtiene de la tabla «clientes» un listado de clientes para ser mostrado en el Tree de la ventana principal.

Parámetros:

estado1 – (_str_): «ABIERTO» o «TOTAL» según se muestren los clientes activos o todos los clientes.

Returns clientes_abiertos o clientes_total:

(_list_) retorna el listado de clientes activos o la totalidad de los clientes dependiendo del parámetro brindado desde el módulo modelo.py.

selecc_para_entrys_sql(id_cliente)[fuente]

Obtiene y retorna de la tabla «clientes» los datos de un cliente requeridos para presentar en los Entrys de la ventana secundaria.

Parámetros:

id_cliente – (_int_) ID del cliente seleccionado.

Returns clientes_actualiza:

(_list_) Listado de atributos del cliente seleccionado para mostrar en los Entrys de la ventana secundaria.

selecc_tree_clientes_sql(id_cliente)[fuente]

Obtiene y retorna de la tabla «clientes» los datos requeridos de un cliente para presentar en el tree listado de la ventana principal.

Parámetros:

id_cliente – (_int_) ID del clientte seleccionado.

Returns cliente_seleccionado:

(_list_) listado de atributos del cliente seleccionado que serán presentados en el Tree de la ventana principal.

selecc_tree_consumos_sql(id_cliente)[fuente]

Obtiene y retorna de la tabla «clientes» los datos requeridos de un cliente para presentar en el tree listado_consumos de la ventana secundaria.

Parámetros:

id_cliente – (_int_) ID del clientte seleccionado.

Returns cliente_seleccionado:

(_list_) listado de atributos del cliente seleccionado que serán presentados en el Tree de la ventana principal.

class admin_bases.BaseConsumos[fuente]

Bases: Conexion

La clase BaseConsumos() cuenta con los métodos que brindan con las directrices necesarias para administrar la información de la base de datos «consumos».

alta_compra_sql(id_cliente, producto, precio, root)[fuente]

Agrega un pedido a la tabla «consumos» y lo asocia al cliente seleccionado mediante su ID.

Parámetros:

  • id_cliente – (_int_) ID del cliente seleccionado.

  • producto – (_str_) nombre del producto.

  • precio – (_float_) precio del producto.

  • root – (_root_) objeto root de Tkinter.

baja_compra_sql(id_cliente, producto, precio, root)[fuente]

Borra un pedido a la tabla «consumos».

Parámetros:

  • id_cliente – (_int_) ID del cliente seleccionado.

  • producto – (_str_) nombre del producto.

  • precio – (_float_) precio del producto.

  • root – (_root_) objeto root de Tkinter.

resta_total_sql(id_client, nuevo_precio, root)[fuente]

Actualiza en la tabla «clientes» el atributo total del cliente seleccionado al borrar un pedido.

Parámetros:

  • id_client – (_int_) ID del cliente seleccionado.

  • nuevo_precio – (_float_) precio del pedido borrado.

  • root – (_root_) objeto root de Tkinter.

suma_total_sql(id_client, nuevo_precio, root)[fuente]

Actualiza en la tabla «clientes» el atributo total del cliente seleccionado al agregar un pedido.

Parámetros:

  • id_client – (_int_) ID del cliente seleccionado.

  • nuevo_precio – (_float_) precio del pedido agregado.

  • root – (_root_) objeto root de Tkinter.

Crea la tabla «consumos».

Crea la tabla «id_base» que proporcionará los números de ID para la tabla «clientes».

get_id()[fuente]

Proporciona un número de ID autoincrementable, lo almacena en la tabla «id_base» y lo retorna para ser utilizado como PRIMARY KEY en la tabla «clientes».

Returns id:

(_int_) integer para ID de la tabla «clientes».
